# Screenshot Checklist — Decision Tree Portfolio Demo

Use this checklist to capture portfolio-ready assets for GitHub, LinkedIn, CV, and personal website.

## Required screenshots

### 1. App landing page

Capture the main Streamlit title and audit note.

Purpose:

```text
Shows the project framing and recruiter-facing explanation.
```

### 2. Input form

Capture the sidebar input form with a realistic sample profile.

Purpose:

```text
Shows that the demo is interactive and easy to test.
```

### 3. Prediction result

Capture the predicted class and test accuracy metric cards.

Purpose:

```text
Shows the classification output clearly.
```

### 4. Prediction probability

Capture the probability table.

Purpose:

```text
Shows model confidence and avoids presenting prediction as absolute truth.
```

### 5. Decision path explanation

Capture the rule-path table.

Purpose:

```text
Shows the explainability value of the Decision Tree model.
```

### 6. Confusion matrix

Capture the confusion matrix tab.

Purpose:

```text
Shows evaluation beyond accuracy.
```

### 7. Classification report

Capture precision, recall, F1-score, and support.

Purpose:

```text
Shows class-level model performance.
```

### 8. Tree preview

Capture the visual Decision Tree preview.

Purpose:

```text
Shows model interpretability and split logic.
```

### 9. Dataset notes

Capture dataset notes showing feature groups and preprocessing summary.

Purpose:

```text
Shows transparency about data and modeling choices.
```

### 10. Sensitive attribute toggle

Capture the toggle that excludes `race` and `sex`.

Purpose:

```text
Shows responsible ML awareness.
```

## Screenshot quality rules

```text
[ ] Crop browser clutter.
[ ] Use consistent window size.
[ ] Avoid exposing local usernames or private folders.
[ ] Avoid showing raw rows of data unnecessarily.
[ ] Use readable zoom level.
[ ] Keep screenshots in PNG format.
[ ] Store screenshots in a dedicated screenshots/ folder.
```
